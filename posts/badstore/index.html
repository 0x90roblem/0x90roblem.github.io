Badstore - VulnHub Solution

6 Jan 2024

New year, new challenge I decided to try VulnHub, it is different from the classic CTF i use to. Where to start? Easy at the beginning, I navigated to the timeline page, scrolled all the way down, found the first released machine, Badstore: 1.2.3, downloaded it, installed it, and let’s get started. Open the other VM with Kali and launch a scan with Nmap (classic), below the command:

sudo nmap -sC -sV -oA badstore/nmap_result 192.168.1.66

This was the output:

The machine expose a web server, this is first page:

I added the badstore.net domain in the /etc/hosts file, in this way i was able to navigate using the hostname instead of IP address.

This is where frustration began. After opening Burp and adding “badstore.net” to the scope, I didn’t know what to search for and where to find it. I examined all of the web requests, but nothing came up. I knew it had to be something simple, because the machine was considered easy. The next day, i re-examined the Nmap scan. I had an idea: maybe the vulnerability is an SQL injection since the machine exposes My-SQL service. Send the login request to the Repeater tab in Burp. Try different payloads, but nothing again. After more than I want to admit attempts i found a valid payload:

admin'  or 'x'=x'

I logged in as ‘Master System Administrator’ and challenge completed.

It took longer than I expected, but it turned out to be quite enjoyable in the end.
